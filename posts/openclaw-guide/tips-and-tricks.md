# OpenClaw — تركات ونصائح ميدانية

## قبل ما تبدأ

**تيليغرام أسرع من واتساب للإعداد الأول.**
واتساب يحتاج QR code ومزامنة الهاتف. تيليغرام bot token جاهز في دقيقتين.

**Node 24 وليس 22.**
Node 22 يشتغل لكن قد تواجه تحذيرات أو مشاكل في بعض الـ plugins. ثبّت 24 مباشرة.

---

## الـ Daemon مهم

```bash
openclaw onboard --install-daemon
```

بدون daemon، الـ gateway يتوقف لما تغلق الـ terminal. مع الـ daemon يشتغل في الخلفية دائماً، حتى بعد إعادة تشغيل الجهاز.

---

## `/compact` يوفّر tokens

لما الجلسة تطول، ارسل `/compact` للـ bot. يضغط السياق ويبقي الأهم فقط — يقلل الاستهلاك بدون تأثير على الجودة.

---

## الـ dmPolicy مربك في البداية

إذا أرسلت للبوت وما ردّ أو طلب "pairing code":
```json
{
  "gateway": {
    "dmPolicy": "open"
  }
}
```
هذا يفتح الـ DM لكل من يعرف الـ bot. إذا الـ bot خاص عليك، آمن. إذا عام، اتركه على `"pairing"`.

---

## ChatGPT subscription: لا تنسى الـ runtime

**الخطأ الأشهر:** تربط OAuth بنجاح لكن ما تحدد `agentRuntime: { id: "codex" }` — الـ agent يعمل بالـ API key path ويفشل.

**الصواب:**
```json
"agentRuntime": { "id": "codex" }
```
هذا السطر يحوّل OpenClaw لاستخدام Codex runtime الأصلي اللي يتعامل مع اشتراكك مباشرة.

## Ollama للتجربة المجانية

```bash
ollama pull llama3.3
```
ثم في `openclaw.json`:
```json
{ "agents": { "defaults": { "model": { "primary": "ollama/llama3.3" } } } }
```
مجاني بالكامل، يعمل offline. مناسب لاختبار OpenClaw قبل تقرر أي مزود تدفع.

---

## واجهة الويب أفضل لإدارة الجلسات

`http://127.0.0.1:18789/` — افتحها من المتصفح على نفس الجهاز. تعطيك رؤية واضحة للجلسات النشطة، الـ logs، والإعدادات. أفضل من تعديل JSON يدوياً للبداية.

---

## اطلب مهام صغيرة أولاً

لا تبدأ بـ "أدر بريدي الوارد كله". ابدأ بـ "لخّص آخر 5 إيميلات". ثم "احذف الـ spam". ثم وسّع الصلاحيات تدريجياً بعد ما تثق بالسلوك.

---

## `/think 2` للمهام المعقدة

```
/think 2
ثم اكتب مهمتك
```
يرفع مستوى التفكير قبل التنفيذ. مفيد لمهام متعددة الخطوات أو اللي تحتاج قرارات.
