# نصائح ميدانية — Local AI مع Claude Code

## 1. اختبر مهمة حقيقية أولاً

لا تحكم على النموذج من محادثة عشوائية. اختبره على مهمة تؤديها يومياً: تفسير ملف، إصلاح خطأ صغير، كتابة اختبار.

## 2. ابدأ بنموذج واحد فقط

اختر `qwen2.5-coder` وأتقن ربطه قبل أن تجرب نماذج أخرى. عشرة نماذج في نفس اليوم = لا قرار.

## 3. خفّض السياق على أجهزة CPU

```bash
OLLAMA_CONTEXT_LENGTH=16000 ollama serve
```

السياق الكبير مع معالج ضعيف يبطئ كل شيء. ابدأ بـ 16K ثم ارفع تدريجياً.

## 4. استخدم مهام صغيرة ومحددة

```text
فسّر هذا الملف فقط. لا تعدل شيئاً. أعطني ملخصاً في 5 نقاط.
```

كلما ضاق النطاق، كانت النتيجة أفضل مع النماذج المحلية.

## 5. راقب النماذج الشغّالة

```bash
ollama ps              # النماذج التي تعمل حالياً
ollama stop MODEL      # أوقف نموذجاً بعينه لتحرير الذاكرة
ollama list            # كل النماذج المحملة
```

## 6. أعد تشغيل Ollama بسياق مختلف عند الحاجة

```bash
# للأجهزة القوية
OLLAMA_CONTEXT_LENGTH=64000 ollama serve
```

يجب إعادة التشغيل لتطبيق التغيير — لا يُطبَّق في الجلسة الحالية.

## 7. احتفظ بـ Claude الأصلي للقرارات الكبيرة

النماذج المحلية للمهام التكرارية والصغيرة. Claude الأصلي للمعمارية والأمان وقرارات الإنتاج.

## 8. لا تثبّت نموذجاً بسبب ضجة التسويق

اختبر على مهمة حقيقية واحدة ثم قرر. الاسم الكبير لا يضمن نتيجة جيدة في تطبيقك المحدد.
