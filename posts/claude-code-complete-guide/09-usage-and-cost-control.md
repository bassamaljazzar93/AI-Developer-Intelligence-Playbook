# 09 — تقليل الاستهلاك والتكلفة

## لماذا يستهلك Claude Code بسرعة؟

Claude Code قد يستهلك أكثر من المحادثة العادية لأنه يعمل بأسلوب agentic:

- يقرأ ملفات.
- يبحث داخل المشروع.
- يقارن نتائج.
- يراجع diff.
- قد يشغل أدوات متعددة.
- قد يحافظ على سياق طويل.

هذا مفيد، لكنه يحتاج إدارة.

## أكبر أسباب الهدر

### 1. طلب واسع جدًا

مثال غير جيد:

```text
افحص المشروع كاملًا وحسّنه.
```

هذا يدفعه لقراءة الكثير بلا هدف واضح.

الأفضل:

```text
افحص جزء authentication فقط واقترح أول 3 تحسينات قابلة للاختبار.
```

### 2. جلسة طويلة بلا تلخيص

الجلسة الطويلة تزيد السياق وتقلل التركيز. اطلب ملخصًا دوريًا:

```text
Summarize the current state, completed work, remaining work, and next exact step.
```

### 3. تنفيذ بدون خطة

عندما يبدأ التنفيذ بدون plan، غالبًا سيجرب أكثر من طريق.

### 4. عدم وجود tests

إذا لا توجد طريقة تحقق، سيكرر المحاولة بالكلام بدل القياس.

### 5. قراءة ملفات كثيرة غير مرتبطة

حدد نطاق القراءة:

```text
Focus only on these files unless you explain why more files are needed.
```

## كيف تقلل الاستهلاك؟

### 1. استخدم Scope واضح

اكتب:

```text
Only inspect files related to the login flow.
```

### 2. اطلب خطة مختصرة

```text
Give me a short plan. Maximum 5 steps.
```

### 3. نفّذ خطوة واحدة

```text
Do step 1 only.
```

### 4. اطلب مخرجات قصيرة

```text
Keep the answer concise. Focus on decisions and next action.
```

### 5. اعتمد commands متكررة

الـ commands تقلل كتابة prompts طويلة وتحافظ على نفس الأسلوب.

## متى تبدأ جلسة جديدة؟

ابدأ جلسة جديدة عندما:

- تغيرت المهمة بالكامل.
- أصبح السياق طويلًا.
- انتهت مرحلة واضحة.
- تريد مراجعة مستقلة.

قبل فتح جلسة جديدة، اطلب ملخص انتقال:

```text
Create a handoff summary for the next Claude Code session.
```

## نموذج Handoff جيد

```md
## Current Goal

## Completed

## Files Changed

## Tests Run

## Known Risks

## Next Step
```

## كيف تعرف أن الاستهلاك غير صحي؟

علامات:

- يقرأ عشرات الملفات بلا سبب.
- يكرر نفس التحليل.
- يقترح حلولًا متغيرة كل مرة.
- يتوسع خارج المهمة.
- يكتب إجابات طويلة بلا قرار.

## قاعدة 24 ساعة

في أول يوم، لا تختبر كل شيء. اختبر 3 مهام فقط:

1. فهم مشروع.
2. إصلاح صغير.
3. كتابة test أو docs.

إذا استهلك كثيرًا ولم يوفر وقتًا، غيّر workflow قبل أن تزيد الاستخدام.

## القرار

تقليل الاستهلاك لا يعني استخدام Claude Code أقل. يعني استخدامه بذكاء: نطاق واضح، خطوات صغيرة، تحقق سريع، وجلسات قصيرة.
