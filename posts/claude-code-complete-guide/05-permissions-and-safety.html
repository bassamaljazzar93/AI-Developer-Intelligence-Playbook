الصلاحيات والأمان التشغيلي

الهدف ليس فتح كل شيء، بل إعطاء Claude Code مساحة عمل كافية مع بقاء كل خطوة قابلة للمراجعة.

مبدأ البداية

Least privilege first. Expand only when needed.

ابدأ بقراءة وخطة. بعد أن يثبت فهمه، اسمح بتعديل محدود.

مستويات الثقة

قراءة فقط

مناسب لأول جلسة أو مشروع جديد.

تعديل محدود

مناسب لملف واحد أو مهمة صغيرة.

Workflow

بعد وجود Git وtests وتعليمات واضحة.

توسيع

فقط بعد نجاح متكرر في مهام صغيرة.

قواعد عملية

  • افصل القراءة عن التنفيذ.
  • راجع الملفات المتغيرة قبل المتابعة.
  • لا تسمح بتغييرات غير مرتبطة.
  • اكتب قواعد المشروع في ملف واضح.

Prompt آمن

Before doing anything, list the files you expect to touch and why. Do not edit yet.
علامة خطر: إذا تغيّرت ملفات كثيرة بلا سبب، أوقف الجلسة واطلب مراجعة diff.
