
## مستويات الاستخدام

### المستوى الأول: قراءة وفهم

- شرح بنية المشروع.
- تلخيص الملفات المهمة.
- تحديد أين توجد أوامر التشغيل والاختبار.

### المستوى الثاني: تعديل محدود

- إصلاح مشكلة صغيرة.
- كتابة اختبار واحد.
- تحسين documentation.

### المستوى الثالث: تنفيذ منظم

- خطة من عدة خطوات.
- تنفيذ تدريجي.
- مراجعة الفرق بعد كل مرحلة.

### المستوى الرابع: طريقة عمل متكررة

- أوامر مخصصة.
- تعليمات مشروع واضحة.
- قوالب prompts.
- سجل مشاكل وحلول.

## معيار النجاح

بعد أي جلسة اسأل:

1. هل وفّر وقتًا؟
2. هل الناتج قابل للاختبار؟
3. هل عدّل ملفات غير ضرورية؟
4. هل احتاج تصحيحًا كثيرًا؟
5. هل يجب تحديث تعليمات المشروع؟

## القاعدة العملية

لا تجعل Claude Code يعمل على مهمة كبيرة قبل أن ينجح في مهمة صغيرة قابلة للقياس.
