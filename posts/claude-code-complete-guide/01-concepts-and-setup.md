# 01 — الفهم العام والتثبيت

## ما هو Claude Code؟

Claude Code هو أداة تطوير تعمل داخل التيرمنال. الفكرة ليست أن تكتب سؤالًا وتحصل على جواب فقط، بل أن يعمل Claude داخل مجلد المشروع، يقرأ الملفات، يقترح تغييرات، ينفذ تعديلات، ويساعدك في الاختبار والمراجعة.

## الفرق بين Claude Code وClaude Chat

| العنصر | Claude Chat | Claude Code |
|---|---|---|
| المكان | المتصفح أو التطبيق | التيرمنال أو IDE |
| الوصول للمشروع | محدود بما ترفعه | يعمل داخل مجلد المشروع |
| التعديل على الملفات | غالبًا يدوي | يمكنه اقتراح أو تنفيذ تغييرات |
| الاستخدام المثالي | تفكير وشرح وكتابة | تطوير عملي داخل codebase |

## الفرق بين Claude Code وAPI

Claude Code ليس مجرد API call. هو طبقة workflow فوق النموذج:

- يفهم بنية المشروع.
- يتفاعل مع الملفات.
- يستخدم أدوات داخلية.
- يتعامل مع git diff.
- يمكن ربطه بأدوات خارجية.

أما API فهو مناسب عندما تبني تطبيقك أنت فوق النموذج.

## متطلبات قبل البدء

قبل استخدام Claude Code في مشروع حقيقي، جهّز التالي:

1. مشروع داخل git.
2. طريقة واضحة للتشغيل.
3. طريقة واضحة للاختبار.
4. ملف تعليمات مختصر للمشروع.
5. فهم للملفات الحساسة التي لا تريد تعديلها.

## التثبيت

راجع دائمًا صفحة setup الرسمية، لأن طرق التثبيت تتغير. المستودع الرسمي يوضح أن بعض الطرق القديمة مثل npm أصبحت غير مفضلة مقارنة بطرق التثبيت الرسمية الجديدة.

### Mac / Linux

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### Windows PowerShell

```powershell
irm https://claude.ai/install.ps1 | iex
```

### Windows WinGet

```powershell
winget install Anthropic.ClaudeCode
```

## أول تشغيل

ادخل إلى مجلد مشروعك:

```bash
cd path/to/project
claude
```

أول طلب لا يجب أن يكون تعديلًا. اجعله قراءة فقط:

```text
Read this project only. Do not edit files. Summarize the architecture, main commands, and possible risks.
```

## أول نتيجة تبحث عنها

إذا كان Claude Code مضبوطًا بشكل جيد، يجب أن يعطيك:

- ملخصًا واضحًا للبنية.
- قائمة ملفات مهمة.
- طريقة تشغيل أو اختبار إن كانت موجودة.
- أسئلة قليلة ومحددة إن كان شيء ناقصًا.
- لا يعدل الملفات إذا طلبت منه عدم التعديل.

## متى لا تبدأ؟

لا تبدأ استخدامه مباشرة في مشروع حساس إذا:

- لا يوجد git clean state.
- لا تعرف كيف تسترجع التغييرات.
- لا توجد طريقة اختبار.
- المشروع يحتوي ملفات حساسة غير موثقة.
- المهمة المطلوبة واسعة جدًا.

## القرار

ابدأ بـ Claude Code كمساعد قراءة وتحليل أولًا. بعد أن يثبت أنه يفهم مشروعك، اسمح له بتعديل محدود وقابل للاختبار.
