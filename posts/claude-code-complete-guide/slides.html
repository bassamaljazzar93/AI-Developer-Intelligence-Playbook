الدليل الكامل لاستخدام Claude Code

دليل عملي مبني من الوثائق الرسمية، GitHub Issues، وملاحظات المجتمع. الهدف أن تستخدم Claude Code كأداة إنتاجية حقيقية لا كتجربة عشوائية.

ابدأREADMEالنصائحالمشاكل
1

ما هو Claude Code؟

Agentic coding tool يعمل من التيرمنال ويفهم مشروعك وينفذ مهام برمجية.

ممتاز في

فهم الكود، refactor، tests، docs، git workflow.

يحتاج ضبط في

المشاريع الكبيرة، الصلاحيات، الجلسات الطويلة، والمهام غير المحددة.

2

البداية السريعة

ثبّت بالطريقة الرسمية ثم ابدأ داخل مجلد مشروع.

cd my-project
claude
لا تبدأ بتعديل. أول طلب: اقرأ المشروع فقط ولا تغيّر ملفات.
3

Workflow الصحيح

Read → Plan → Execute → Verify → Review.

Plan first

لأي مهمة فيها أكثر من خطوة، اطلب خطة قبل التنفيذ.

Small steps

نفّذ خطوة واحدة، ثم اختبر، ثم أكمل.

5

الصلاحيات

ابدأ محدودًا ثم وسّع حسب الحاجة.

لا تجعل كل شيء مسموحًا من أول يوم. الصلاحيات الواسعة توفر وقتًا لكنها تزيد خطر التعديل الخاطئ.
6

Plugins & Skills

استخدمها عندما يتكرر نفس workflow.

Slash commands

أوامر مختصرة مثل review أو test-plan.

Skills

معرفة قابلة لإعادة الاستخدام في مهام متخصصة.

7

MCP & Hooks

MCP يربط أدوات خارجية، وHooks تضيف أتمتة عند أحداث معينة.

MCP

GitHub، Filesystem، Database، Browser، Memory.

Hooks

تحقق قبل أو بعد استخدام أداة، أو عند نهاية المهمة.

8

تركات ونصائح

أفضل نصيحة: لا تطلب تنفيذًا كبيرًا مرة واحدة.

بعد أي خطأ متكرر، اطلب منه تحديث تعليمات المشروع حتى لا يكرر الخطأ.
