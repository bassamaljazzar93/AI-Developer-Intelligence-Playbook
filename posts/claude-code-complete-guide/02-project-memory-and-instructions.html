Chapter 02

إعداد المشروع والذاكرة

إذا لم تكتب قواعد المشروع، سيحاول Claude Code الاستنتاج. أحيانًا ينجح، وأحيانًا يكرر نفس الأخطاء.

لماذا تحتاج تعليمات؟

  • تحديد طريقة التشغيل والاختبار.
  • توضيح style المشروع.
  • حماية الملفات الحساسة أو generated.
  • تقليل تكرار نفس التعليمات في كل جلسة.

CLAUDE.md

ملف خاص بتعليمات Claude Code داخل المشروع.

# Project Instructions

## Run
Use the documented command in README.

## Testing
Before saying done, run the smallest relevant test when possible.

## Style
Keep changes minimal. Do not reformat unrelated files.

## Safety
Do not add secrets. Do not edit generated files unless asked.

AGENTS.md

مفيد عندما تريد قواعد عامة لأي coding agent، وليس Claude فقط.

# Agent Rules

- Read before editing.
- Prefer small changes.
- Explain risk before large changes.
- Do not claim tests passed unless actually run.

ماذا تكتب؟

مهم

وصف المشروع، أوامر التشغيل، أوامر الاختبار، الملفات المهمة، والممنوعات.

تجنب

تعليمات عامة مثل “اكتب كود جيد”. اكتب قواعد قابلة للتحقق.

القرار: أي مشروع ستستخدم معه Claude Code أكثر من مرة يحتاج ملف تعليمات مختصر.
