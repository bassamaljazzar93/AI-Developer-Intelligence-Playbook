# 06 — Hooks والأتمتة

## ما هي Hooks؟

Hooks هي طريقة لإضافة سلوك تلقائي حول جلسة Claude Code أو حول استخدام الأدوات. الفكرة أن تجعل بعض القواعد تعمل تلقائيًا بدل تذكير Claude بها كل مرة.

## متى تستخدم Hooks؟

استخدمها عندما تريد:

- تذكير ثابت في بداية الجلسة.
- فحص نمط متكرر قبل إجراء معين.
- تنبيه بعد انتهاء مهمة.
- منع سلوك يتكرر كثيرًا.
- جمع ملاحظات تشغيلية حول الجلسات.

## أمثلة استخدام عملية

### 1. تذكير بداية الجلسة

الفكرة:

- اقرأ تعليمات المشروع.
- لا تبدأ بتغييرات كبيرة.
- لخص الخطة قبل التنفيذ.

### 2. تذكير قبل التعديل

الفكرة:

- هل قرأت الملف؟
- هل التغيير مرتبط بالمهمة؟
- هل يوجد اختبار؟

### 3. تذكير نهاية المهمة

الفكرة:

- ما الملفات التي تغيرت؟
- هل تم التحقق؟
- ما المخاطر المتبقية؟

## متى لا تستخدم Hooks؟

لا تستخدم Hooks فقط لأنك تستطيع. استخدمها لحل مشكلة متكررة.

أمثلة على استخدام غير مفيد:

- Hook لكل خطوة صغيرة.
- قواعد طويلة جدًا.
- رسائل مزعجة تستهلك التركيز.
- أتمتة لا تفهمها جيدًا.

## قاعدة بناء Hook جيد

Hook جيد يجب أن يكون:

1. قصيرًا.
2. واضح الهدف.
3. مرتبطًا بمشكلة متكررة.
4. لا يضيف ضوضاء كثيرة.
5. قابلًا للإيقاف أو التعديل.

## أمثلة مشاكل تستحق Hook

- Claude ينسى ذكر الاختبارات.
- Claude يغير ملفات غير مرتبطة.
- Claude يعلن الانتهاء بدون تحقق.
- Claude يتجاهل تعليمات المشروع.
- Claude يكتب documentation لا تطابق الكود.

## الفرق بين Hook وCommand

| العنصر | Hook | Command |
|---|---|---|
| التشغيل | تلقائي أو حسب حدث | يدوي عند الطلب |
| الاستخدام | قواعد متكررة | مهام متكررة |
| مثال | تذكير قبل نهاية الجلسة | `/review` |

## أفضل بداية

ابدأ بدون Hooks. بعد أسبوع من الاستخدام، راقب الأخطاء المتكررة. إذا تكرر نفس الخطأ ثلاث مرات، حوله إلى قاعدة أو Hook.

## القرار

Hooks مفيدة عندما تحول الخبرة المتكررة إلى حماية تلقائية. لا تبدأ بها مبكرًا قبل فهم مشاكلك الحقيقية مع Claude Code.
