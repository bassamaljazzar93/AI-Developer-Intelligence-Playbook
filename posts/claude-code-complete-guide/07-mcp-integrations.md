# 07 — MCP والربط مع الأدوات

## ما هو MCP؟

MCP يعني Model Context Protocol. فكرته أن يتيح لـ Claude Code الوصول إلى أدوات أو مصادر خارجية بطريقة منظمة.

بدل أن يكون Claude محصورًا في المحادثة، يمكنه استخدام أدوات مثل:

- ملفات المشروع.
- GitHub.
- قواعد بيانات.
- browser/search tools.
- أنظمة ذاكرة.
- أدوات داخلية للشركة.

## لماذا MCP مهم؟

لأن جودة Claude Code تعتمد على السياق. إذا كان السياق ناقصًا، ستكون النتيجة ناقصة. MCP يساعد في توفير السياق أو تنفيذ عمليات محددة.

## أمثلة عملية

### GitHub MCP

مفيد لـ:

- قراءة Issues.
- مراجعة Pull Requests.
- البحث في الكود.
- إنشاء تعليقات أو تلخيصات.

### Filesystem MCP

مفيد لـ:

- قراءة ملفات خارج مجلد المشروع.
- تنظيم مصادر معرفة.
- البحث في أدلة محددة.

### Database MCP

مفيد لـ:

- فهم schema.
- كتابة queries.
- تحليل بيانات تطوير.

لا تستخدمه على production data بلا ضوابط واضحة.

### Browser/Search MCP

مفيد لـ:

- قراءة docs حديثة.
- مقارنة أدوات.
- فحص أمثلة من المجتمع.

## كيف تختار MCP مناسب؟

اسأل هذه الأسئلة:

1. هل يحل مشكلة متكررة؟
2. هل يقلل النسخ واللصق؟
3. هل يعطي سياقًا أفضل؟
4. هل يمكن ضبط حدوده؟
5. هل تثق بالمصدر؟
6. هل يحتاج بيانات حساسة؟

## ترتيب البدء المقترح

ابدأ بهذه البساطة:

1. GitHub أو filesystem.
2. Docs/search.
3. Database للتطوير فقط.
4. أدوات داخلية لاحقًا.

## مخاطر MCP

MCP قوي، لكنه يزيد سطح الخطأ. انتبه إلى:

- مشاركة بيانات غير مقصودة.
- وصول أوسع من اللازم.
- أدوات غير موثوقة.
- أوامر غير مفهومة.
- نتائج قديمة أو غير دقيقة.

## قاعدة ذهبية

لا تضف MCP لأنه مشهور. أضفه فقط عندما يكون لديك use case واضح.

## مثال Use Case جيد

المشكلة:

كل مرة تطلب من Claude مراجعة Issue في GitHub، تنسخ النص يدويًا.

الحل:

GitHub MCP أو connector يسمح له بقراءة issue مباشرة.

الفائدة:

- سياق أفضل.
- وقت أقل.
- أخطاء نسخ أقل.

## مثال Use Case سيء

المشكلة:

تريد تجربة كل MCP موجود.

النتيجة:

- إعداد معقد.
- أخطاء أكثر.
- صعوبة معرفة سبب المشكلة.

## القرار

MCP هو طبقة قوة إضافية. لا تبدأ به في اليوم الأول. ابدأ بـ Claude Code الأساسي، ثم أضف MCP عندما تعرف بالضبط ما السياق أو الأداة التي تحتاجها.
