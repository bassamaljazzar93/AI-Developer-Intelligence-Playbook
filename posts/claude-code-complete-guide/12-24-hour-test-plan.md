# 12 — خطة اختبار 24 ساعة

هدف هذه الخطة أن تقرر بسرعة هل Claude Code مفيد لمشروعك أم لا.

لا تختبر كل شيء. اختبر مهام صغيرة تقيس القيمة الحقيقية.

## قبل البدء

جهّز:

- مشروع صغير أو متوسط.
- git branch نظيف.
- أمر تشغيل واضح.
- أمر اختبار إن وجد.
- ملف تعليمات بسيط للمشروع.

## الساعة 1 — فهم المشروع

اطلب:

```text
Read this project only. Do not edit files. Summarize architecture, key files, run command, test command, and first safe improvement.
```

### معيار النجاح

- يذكر ملفات حقيقية.
- لا يخترع أوامر غير موجودة.
- يعطي ملخصًا مفيدًا.
- لا يعدل ملفات.

## الساعة 2 — خطة مهمة صغيرة

اختر مهمة صغيرة مثل:

- تحسين README.
- إصلاح bug واضح.
- إضافة validation.
- كتابة test بسيط.

اطلب:

```text
Create a plan for this small task. Do not edit yet. Include files, risks, and verification.
```

### معيار النجاح

- الخطة قصيرة.
- لا يقترح إعادة بناء كبيرة.
- يحدد verification.

## الساعة 3 — تنفيذ خطوة واحدة

اطلب:

```text
Implement only the first step. Keep changes minimal. Summarize changed files after finishing.
```

### معيار النجاح

- تغييرات قليلة.
- لا يلمس ملفات غير مرتبطة.
- يشرح ما فعل.

## الساعة 4 — التحقق

اطلب:

```text
Verify the change. If you cannot run the test, explain the exact manual check.
```

### معيار النجاح

- لا يدعي نجاحًا غير مثبت.
- يعطي أمر اختبار أو خطوات يدوية.
- يميز بين ما تم تشغيله وما لم يتم.

## الساعة 5 — مراجعة صارمة

اطلب:

```text
Review the current diff as a strict reviewer. Do not edit. Identify bugs, risks, and missing tests.
```

### معيار النجاح

- يعطي ملاحظات مفيدة.
- لا يكرر الكلام العام.
- يميز المخاطر حسب الأولوية.

## الساعة 6 — تحديث التعليمات

اطلب:

```text
Suggest a small update to the project instructions based on this session.
```

### معيار النجاح

- يقترح قاعدة قصيرة.
- القاعدة تمنع خطأ حقيقيًا.
- لا يكتب تعليمات طويلة بلا فائدة.

## نهاية اليوم — قرار

اسأل نفسك:

| السؤال | نعم/لا |
|---|---|
| هل فهم المشروع بسرعة؟ |  |
| هل التزم بعدم التعديل عندما طلبت؟ |  |
| هل نفذ تغييرًا صغيرًا بنجاح؟ |  |
| هل ساعد في الاختبار؟ |  |
| هل قلل وقتك؟ |  |
| هل احتاج إشرافًا مفرطًا؟ |  |
| هل كرر أخطاء خطيرة؟ |  |

## قرار الاستخدام

### استخدمه يوميًا إذا:

- وفّر وقتًا واضحًا.
- التزم بالنطاق.
- ساعدك في الاختبار والمراجعة.
- لم يسبب تغييرات عشوائية.

### استخدمه بحذر إذا:

- يحتاج إشرافًا مستمرًا.
- يقرأ كثيرًا بلا إنجاز.
- يتوسع خارج المهمة.

### أوقفه مؤقتًا إذا:

- يعدّل ملفات حساسة بلا طلب.
- لا يلتزم بالتعليمات.
- لا تستطيع مراجعة تغييره.
- يزيد وقتك بدل تقليله.

## الخلاصة

Claude Code يجب أن يثبت قيمته خلال يوم واحد في مهمة صغيرة. إذا لم ينجح في الصغير، لا تعطه الكبير.
